# 快速使用指南

## 启动项目

1. 在 `numerical-visualizer` 目录下运行：
```bash
npm run dev
```

2. 在浏览器中打开显示的 URL（通常是 http://localhost:5173）

## 快速上手

### 第一步：查看两种设计风格的 Demo
在项目根目录下有两个 HTML 文件：
- `demo-scientific.html` - 科学仪器风格
- `demo-modern.html` - 现代数字艺术风格

直接用浏览器打开这两个文件，查看完整的设计效果，帮助你选择喜欢的风格。

### 第二步：启动 React 应用
进入 `numerical-visualizer` 目录，运行：
```bash
npm run dev
```

应用会在 http://localhost:5173 启动

### 第三步：切换主题
在应用的右上角，点击主题切换按钮：
- 🎨 现代风格 - 浅色渐变，柔和色彩
- 🔬 科学仪器 - 深色主题，霓虹效果

你的主题偏好会自动保存到浏览器的 localStorage。

### 第四步：选择算法并开始计算
1. 点击左侧面板的算法按钮（二分法、牛顿法、埃特肯法、弦截法）
2. 选择预设函数或输入自定义表达式
3. 调整参数（初值、误差、最大迭代次数）
4. 算法会自动运行并显示结果

### 第五步：观察可视化
- **函数曲线图** - 显示函数和迭代点
- **收敛速度图** - 对数坐标显示误差下降
- **数据表格** - 详细的迭代数据
- **收敛盆分析** - 不同初值的收敛情况

### 第六步：使用动画控制器
- ⏮ 跳到第一步
- ⏪ 上一步
- ▶ 播放动画
- ⏸ 暂停
- ⏩ 下一步
- ⏭ 跳到最后一步
- 调整速度滑块（0.1x - 3.0x）

### 第七步：导出数据
在左侧面板底部，点击：
- 📊 导出 CSV - 保存为 CSV 格式
- 📄 导出 JSON - 保存为 JSON 格式

## 示例：使用牛顿法求 √2

1. 选择算法：**牛顿法**
2. 选择函数：**x² - 2**
3. 设置初值：**1.5**
4. 点击播放按钮观察迭代过程
5. 查看收敛速度分析，注意二次收敛的特性

## 示例：比较不同算法

你可以尝试用不同算法求解同一个方程，观察：
- **二分法**：稳定但慢，线性收敛
- **牛顿法**：快速，二次收敛
- **弦截法**：不需要导数，超线性收敛
- **埃特肯法**：加速收敛序列

## 自定义函数示例

在函数输入框中，你可以输入各种数学表达式：

- `x^2 - 4` - 求 ±2
- `sin(x) - 0.5` - 求 sin(x) = 0.5
- `exp(x) - x - 2` - 超越方程
- `x^3 - 2*x - 5` - 三次方程
- `log(x) - 1` - 求 e
- `sqrt(x) - 2` - 求 4

## 提示和技巧

1. **初值选择很重要**：
   - 牛顿法对初值敏感，选择接近根的初值
   - 二分法需要函数在区间端点异号
   - 弦截法需要两个不同的初值

2. **观察收敛速度**：
   - 查看收敛速度图的斜率
   - 二次收敛的误差下降更陡峭

3. **收敛盆分析**：
   - 颜色越深表示收敛越快
   - 半透明区域表示不收敛或超出迭代限制

4. **数据导出**：
   - CSV 格式适合 Excel 分析
   - JSON 格式保留完整的元数据

## 常见问题

**Q: 为什么牛顿法报错？**
A: 可能是导数为零或初值选择不当。尝试更换初值或使用弦截法。

**Q: 为什么收敛盆计算很慢？**
A: 收敛盆需要对大量初值进行计算，这是正常的。分辨率已经优化过，通常几秒内完成。

**Q: 如何保存可视化图像？**
A: 使用浏览器的截图功能，或右键点击图表选择"另存为图片"。

**Q: 可以添加更多算法吗？**
A: 当然！项目采用模块化设计，在 `src/algorithms/` 目录下添加新算法即可。

## 性能优化

如果遇到性能问题：
1. 减少最大迭代次数
2. 增大容许误差
3. 暂时关闭收敛盆分析（它计算量最大）
4. 使用生产版本（`npm run build`）

## 技术支持

遇到问题？
1. 查看浏览器控制台的错误信息
2. 确保使用现代浏览器（Chrome, Firefox, Safari, Edge）
3. 检查 Node.js 版本 >= 16
